JVM 
- implementation of a vm which executes a Java program
- JVM (interprets) -------> bytecode
- stores class info in memory area.
- executes bytecode generated by java compiler.
- manipulates various memory areas at runtime

- COMPONENTS
	- Class Loaders
		- loads, verifies and links bytecode
	- Runtime Data Areas (defined by JVM)
		- memory areas to execute a Java program
		- used during runtime
		- some created on JVM startup and destroyed when JVM exits
		- some created and destroyed with thread
		- Method Area
			- stores runtime constant pools, field, method data
			- code for methods and constructors
			- fully qualified class names
			- created when JVM starts
			- all JVM threads share this mem area
		- Heap Area
			- memory allocated by JVM for all class instances and arrays
			- GC reclaims heap memory for objects
			- Eden Space
				- part of Young Generation space
				- JVM allocates memory from when object is created
			- Survivor Space
				- part of Young Generation space
				- contains existing objects that survived minor GC
			- Tenured Space
				- Old Generation space
				- holds long surviving objects
				- moved here when young generation objects meet threshold
		- Stack Area
			- stores data as frames and each frame stores local variables, partial results and nested method calls
			- created by JVM whenever new thread is created
			- private for each thread
			- entries in the stack are called stack frames
			- frame contains 3 parts
				- Local Variable Array
					- contains all local variables and parameters of the method
				- Operand Stack
					- workspace for storing intermediate calculations result
				- Frame Data
					- stores partial results
					- stores return values for methods
					- stores reference to the Exception table
		- PC Registers
			- each JVM thread has a separate PC Register
			- stores the address of currently executing instruction
		- Native method stacks
			- written in languages other than java
			- also knows as C stacks
			- store native method information
			- used to keep track of the state of native methods whenever they compile into machine codes
			- created by JVM to create a new thread
	- Execution Engine
		- Interpreter
			- executes the instructions using information present in memory areas
			- executes the loaded and verified bytecode
			- when one method is called multiple time, new interpretation is required every time (uses JIT compiler to mitigate this)
		- Just-in-Time (JIT) Compiler
			 - compiles bytecode of the often-called methods into native code at run-time
			 - responsible for the optimization of Java programs
		- Garbage Collector
			- used for memory management
			- deleting unused objects by looking at heap memory
			- is a daemon thread
			- can be called explicitly
				- using Systemgc() method
			- decided by JVM on when to invoke
	- Java Native Interface
		- acts as an interface between Java code and the native C/C++ libraries
		- enables native methods to call code running in the JVM
	- Native Libraries
		- platform specific libraries
		- contains implementation of native methods


JRE
- bindle of software components used to run Java applications
- Core Components
	- an implementation of JVM
	- classes required to run Java programs
	- property files
- Bootstrap Classes
	- found under jre/lib/
	- includes
		- runtime classes in rt.jar
		- internationalization classes in i18n.jar
		- character conversion classes in charsets.jar
		- loaded when JVM starts up
- Extension Classes
	- found in jre/lib/extn/ (extension classpath)
	- contains JavaFX runtime libraries in jfxrt.jar
	- contains locale data for java.text and java.util packages in localedata.jar
	- can add custom jars in this directory
- Property Settings
	- used to maintain config
	- located in /jre/lib/
		- calendar.properties
		- logging.properties
		- net.properties
		- /jre/lib/deploy/
		- /jre/lib/management/
		- jre/lib/security
		- jre/lib/applet
		- jre/lib/fonts

JDK (Java Development Kit)
- provides environment and tools for developing, compiling, debugging and executing a java program
- Components
	- JRE
	- Development Tools
		- Basic Tools
			- utilities for compiling, debugging, archiving, generating Javadocs, etc
			- javac
				- reads class and interface definitions and compiles them into class files
			- java
				- launches java application
			- javadoc
				- generates HTML pages of API documentation from Java source files
			- apl
				- finds and executes annotation processors based on the annotations present in the set of specified source files
			- appletviewer
				- enables to run Java applets without a web browser
			- jar
				- packages java applets/applications into a single archive
			- jdb
				- used to find and fix bugs in Java applications
			- javah
				- produces C header and source files from a Java class
			- javap
				- disassembles and displays info about fields, constructors and methods in a class file
			- extcheck
				- detects version conflicts btw target Java Archive(JAR) file and currently installed extension JAR files
